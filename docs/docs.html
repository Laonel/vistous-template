Structure

Vistous is HTML template. To work properly it requires css, fonts, js and php folders placed in template's root directory. Your html files should be placed in root directory.

CSS files

Template's core CSS code is included in the style.css file. This file is required and needs to be included in all of your html files. File margins.css contains spacing classes, refer to Spacing section of this documentation for more information.

JS files

Vistous template has 3 JavaScript files which are necessary: plugins.js, elements.js and functions.js. The other files are plugins and can be omitted when you know that you won't be using particular plugin.

Grid

Vistous is bases on Bootstrap 3.3 and uses the default Bootstrap 12 column grid system. Columns should be wrapped in both container and row element.

Spacing

Vistous comes with helper classes to space elements out from one another on the page. You can use margin or padding to space elements. Here is how you can use these classes:

<h1 class=”mb-60”>Heading Tag</h1>
<div>
    …
</div>

This will result in heading being 60 pixels above the div element.

The general structure of these classes is as follows: m – for margin, or p – for padding, followed by direction (t – top, r – right, b – bottom, l – left) or nothing to add space around the element.

Example classes:

.mt-0 { margin-top: 0 !important; }
.mt-5 { margin-top: 5px !important; }
.mt-10 { margin-top: 10px !important; }
...
.mt-200 { margin-top: 200px !important; }

.pt-0 { padding-top: 0 !important; }
.pt-5 { padding-top: 5px !important; }
.pt-10 { padding-top: 10px !important; }
...
.pt-200 { padding-top: 200px !important; }

.m-0 { margin: 0 !important; }
.m-20 { margin: 20px !important; }

Header

Header can contain 3 main sections: topbar, bottombar and main header area. Each of these sections can have it's separate layout.

Example header containing one section with 2 columns:

<header class="header">
    <div class="main-header">
        <div class="header-container container">

            <div class="header-row">

                <div class="header-column">

                </div> <!-- end .header-column -->

                <div class="header-column">

                </div> <!-- end .header-column -->

            </div> <!-- end .header-row -->

        </div> <!-- end .header-container -->
    </div> <!-- end .main-header -->
</header><!-- end .header -->

In the example above layout is based on table, which means that each column have width dependent on its content. You can also use bootstrap grid system inside header's sections.

Common header classes:
  • .header-overlay - display header with absolute position, causing it to overlay site's content.
  • .dropdown-dark - change navigation dropdown style to dark.
  • .no-border - hide header's default bottom border.
  • .sticky - make header stick to the top of the browser window (this class activates JavaScript plugin, see below).
  • .sticky-condensed - make header thinner when sticked.
Sticky Header

To activate sticky header functionality add sticky class to the header element. You can also use data-plugin-sticky-header attribute. To set plugin's options add data-plugin-options attribute to the header element. Options are passed in the format of object literal.

<div class="main-header sticky" data-plugin-options='{"classesAdd": "bg-dark"}'>
</div>

This will set the plugin to add .bg-dark class when header is sticked.

Available options
  • classesAdd - list of classes to add when header is sticked.
  • classesRemove - list of classes to remove when header is sticked.
  • alwaysStickOnMobile - whether the header should be always sticked on mobile devices.
Logo

Your logo image or text should be placed inside logo-wrapper element.

<div class="logo-wrapper">
    Your logo here							
</div>

It is also possible to have your logo image change depending on current background.

<div class="logo-wrapper">
    <img class="logo-dark" src="path/to/dark-image">	
    <img class="logo-light" src="path/to/light-image">					
</div>

In the example above logo-light element will be displayed if one of it's parent have .font-white class, logo-dark element will be displayed otherwise. For more information about .font-white class and backgrounds refer to Backgrounds section.

Navigation

To create primary navigation for your site create unlisted list with class .main-nav and wrap it in nav-wrapper element. To activate JavaScript plugin for navigation add data-plugin-main-nav element to nav-wrapper element.

<div class="nav-wrapper" data-plugin-main-nav>

    <ul class="main-nav">
        <li><a href="#">Item 1</a></li>
        <li><a href="#">Item 2</a></li>
        <li><a href="#">Item 3</a></li>
    </ul>

</div>

To create dropdown menu add .dropdown class to li element. Inside it create unlisted list with .dropdown-menu class.

<div class="nav-wrapper" data-plugin-main-nav>

    <ul class="main-nav">
        <li>
            <a href="#">Dropdown</a>

            <ul class="dropdown-menu">
                <a href="#">Item 1</a>
                <a href="#">Item 2</a>
            </ul>
        </li>
        <li><a href="#">Item 2</a></li>
        <li><a href="#">Item 3</a></li>
    </ul>

</div>
Mega Dropdown

Vistous template includes mega dropdown which is of the width of container element. Mega dropdown has following structure: 

<div class="nav-wrapper" data-plugin-main-nav>

    <ul class="main-nav">
        <li class="dropdown dropdown-mega sf-mega">
            <a href="#">Mega Dropdown</a>

            <ul class="dropdown-menu">
                <li>
                    <div class="dropdown-mega-content">

                    </div>
                </li>
            </ul>
        </li>
    </ul>

</div>

Inside dropdown-mega-content element you can add every type of content that you want. You can also use bootstrap grid system. Following code creates multi-column navigation inside mega dropdown: 

<div class="dropdown-mega-content">
    <div class="row">
        <div class="col-md-3">
            <ul class="dropdown-mega-submenu">
                <li></li>
            </ul>
        </div>

        <div class="col-md-3">
            <ul class="dropdown-mega-submenu">
                <li></li>
            </ul>
        </div>
    </div>
</div>

Sliders

Vistous uses Owl Carousel jQuery plugin to handle all slider behaviour. There are several types of preconfigured sliders. The general slider structure is as follows: 

<div class="slider-item">
    
    <div>
    </div>

    <div>
    </div>

</div>

Every immediate child of slider's element is slider's item.

Available slider types:
  • .slider-item - generic slider with responsive functionality. By default displaying 4 elements at the time.
  • .slider-item-small - same as above but displaying 6 elements.
  • .slider-item-large - displaying 3 elements at the time.
  • .slider-item - generic slider displaying single item at the time.
  • .slider-gallery - slider configured to serve as photo gallery. Navigation buttons by default.
  • .slider-fullwidth - slider for displaying fullwidth slides. Navigation buttons by default.

Backgrounds

Vistous template includes classes for manipulating element's background color. These classes are:

  • .bg-white
  • .bg-light
  • .bg-dark

When an element has dark background color, add .font-white class to make its content display properly.

Background Image

To add background image to the element all you need to do is specify data-background-image attribute with path to the image you want to display.

<section class="page-section" data-background-image="path/to/background/image">
</section>
Image Overlay

You can add partly transparent color overlay to make contents of the element more readable. Example classes:

  • .bg-dark-alpha-60 - dark background that is visible in 60% (multiples of 10)
  • .bg-light-alpha-60 - light background that is visible in 60% (multiples of 10)
  • .bg-white-alpha-fade-top - white background that fades to the top
  • .bg-light-alpha-fade-top - light background that fades to the top
  • .bg-dark-alpha-fade-top - dark background that fades to the top
  • .bg-white-alpha-fade-bottom - white background that fades to the bottom
  • .bg-light-alpha-fade-bottom - light background that fades to the bottom
  • .bg-dark-alpha-fade-bottom - dark background that fades to the bottom
Parallax

By default background images are static, which means that they stay in the same place as user scroll the webpage. You can add parallax effect by using following classes:

  • .parallax-1 - the background image scrolls 10 times slower than the webpage
  • .parallax-2 - the background image scrolls 5 time slower than the webpage
  • .parallax-5 - the background image scroll half as fast as the webpage

Available classes for parallax are .parallax-1 up to .parallax-9.

You can add .bg-scroll class to the element to make its background image scroll together with the webpage.

Background Image Position

You can change background image position by using CSS classes:

  • .bg-left-top - Displays image from top left corner.
  • .bg-left-center - Displays image from the left, centered vertically.
  • .bg-left-bottom - Displays image from bottom left corner.
  • .bg-right-top - Displays image from top right corner.
  • .bg-right-center - Displays image from the right, centered vertically.
  • .bg-right-bottom - Displays image from bottom right corner.
  • .bg-center-top - Displays image from the top, centered horizontally.
  • .bg-center-center - Displays image centered.
  • .bg-center-bottom - Displays image from the bottom, centered horizontally.
Background Slideshow

With Vistous you can create background image slideshows. To initiate plugin add data-plugin-bg-slideshow attribute to the element you want to add slideshow to. Slides are specified in data-slides attribute as an array of object literals.

<section data-plugin-bg-slideshow data-slides='[{"src": "path/to/first/slide/image"}, {"src": "path/to/second/slide/image"}, {"img": "path/to/third/slide/image"}]'>
</section>
Background Video

Vistous template supports playing YouTube videos in background. To play a video as a background of an element, you just need to add the player element inside it and specify plugin's options.

<section id="home">
    <div class="player" data-property="{videoURL:'https://youtu.be/p91tO9Ve9kk',containment:'#home',autoPlay:true, showControls:false, showYTLogo: false, mute:true, startAt:0, opacity:1}">
    </div>
</section>

YTPlayer jQuery plugin is used to play videos, refer to documentation for more information about the plugin.

Side Panel

Vistous features customizable side panel which you can add to your webpages. To create side panel you need two things: trigger element and the actual side panel element. Trigger is usually a button or link that causes side panel to appear when clicked.

<a href="#sidepanel" class="sp-open-trigger">
    <i class="icon_menu"></i>
</a>

<div class="side-panel" id="sidepanel" data-plugin-options='{"type": "overlay", "side": "left", "allowScroll": "true"}'>

    <div class="sp-wrapper">

    	Here add your content

    </div>

</div>

You need to initialize side panel plugin by adding .side-panel class or data-plugin-side-panel attribute to the side panel element. You can set plugin's options through data-plugin-options attribute, like in the example above.

Available options:
  • type - specifies how the side panel will display (default: overlay)
    • overlay - side panel displays over webpage's content, overlaying it
    • push - side panel "pushes" webpage's content when displayed
  • side - specifies on which side of the page will the side panel be displayed (left or right, default: left)
  • allowScroll - whether the side panel's content should be scrollable (default: false)
  • closeOnBgClick - whether the side panel should be closed when user clicks somewhere outside its area (default: true)

Portfolio

Vistous template has robust and advanced portfolio system, that include different grid types, filtering and item popup.

<div class="portfolio" data-plugin-options='{"filters": {"enabled": true}}'>

    <div id="portfolio-filters" class="filters filters-center">		
        <div class="filters-container">
            <ul>
                <li class="filter filter-btn active" data-filter="all"><span>All</span></li>
            </ul>
        </div>
    </div>

    <div class="portfolio-container portfolio-col-2 portfolio-no-space">

    </div>

</div>

Example structure of the portfolio element. Portfolio's parent element should have .portfolio class. Filters and portfolio items are added inside this element. Above code will result in regular grid 2 column portfolio without spaces between items and enabled filtering.

Portfolio classes:
  • .portfolio-col-2 - 2 column portfolio, single column on screens smaller than 768 px.
  • .portfolio-col-3 - 2 columns on screens wider than 768 px, 3 columns on screens wider than 992 px.
  • .portfolio-col-4 - 2 columns on screens wider than 768 px, 4 columns on screens wider than 992 px.
  • .portfolio-col-5 - 2 columns on screens wider than 768 px, 4 columns on screens wider than 992 px, 5 columns on screens wider than 1200px.
  • .portfolio-wide - spread portfolio items through all of the webpage width.
  • .portfolio-no-space - no spaces between portfolio items.
Portfolio plugin options:
  • listenOn - DOM element on which the plugin wil be listening for events.
  • layoutMode - portfolio items layout mode (currently available values: masonry or grid).
  • itemContainer - selector for the element that will hold all of the portfolio items.
  • masonry - settings for the masonry layout mode (array).
  • popup - settings for the portfolio item popup (array).
    • enabled - true to enable popup functionality, false to disable it.
    • selector - selector for the element that will toggle the popup.
    • template - HTML markup template of the popup element.
    • ajaxError - error message to display when there is an AJAX loading error.
    • onLoad - callback function that is called on item loading.
    • onSuccess - callback function that is caled when item loading was successful.
    • onError - callback function that is called when there was an error during item loading.
  • filtering - settings for the portfolio item filtering (array).
    • enabled - true to enable filtering, false to disable it.
    • container - selector for the filtes container element.
    • selector - selector for the filter element.
Portfolio Gallery

You can create lightbox gallery from your portfolio items images. Just wrap portfolio items in a div element and activate lightbox plugin.

<div class="lightbox-gallery" data-plugin-options='{"delegate": ".lightbox-item", "type": "image", "gallery": {"enabled": true}}'>>

    <div class="portfolio-item">
        <div class="pi-image">

            <a class="lightbox-item" href="path/to/your/image"></a>
            <img src="path/to/your/image">

        </div>
    </div>

</div>

In the above example every element with class .lightbox-item gets added to the gallery. The image file should be specified in href attribute of the .lightbox-item element. For more information and examples refer to the Lightbox Element section of this file and demo files.

Blog

Each blog entry in Vistous template is contained inside element with .entry class. The structure of such element is presented in following example.

<div class="entry">

    <div class="entry-image">
        <img src="#" alt="text">
    </div>

    <h5 class="entry-title">Entry Title</h5>

    <ul class="entry-meta">
        <li>
            <i class="fa fa-user icon-small"></i>
            John Doe
        </li>
    </ul>

    <p>Sed dapibus nibh et diam scelerisque condimentum.</p>

    <p><a href="#" class="entry-read-more">Read More</a></p>

</div> <!-- end .entry -->

Blog entries should be wrapped by an .blog-content element. Vistous also supports sidebars. To create a sidebar just add new element with .sidebar class after .blog-content element. You can change position of the sidebar by adding .left or .right classes to it and to .blog-content element. The width of blog-content and sidebar is controlled by Bootstrap grid. Below is basic example of blog layout.

<div class="container">
    <div class="row">

        <div class="col-md-9 col-sm-12 blog-content right">
        </div>

        <div class="col-md-3 col-sm-12 sidebar left">
        </div>

    </div>
</div>

This will create 9 columns wide blog content and 3 columns wide sidebar, on devices wider than 991 px. Fullwidth content and sidebar on smaller devices. Sidebar will be on the left side.

Refer to demo files for more information and examples about blog system in Vistous.

Widgets

Vistous include HTML widgets that are quick way to add particular content to your website. Below is markup and details for each of the default widgets.

<div class="widget">

    <h5 class="widget-title">Widget Title</h5>

    <div class="widget-text">
    </div>

</div>

Standard markup for widgets. You need to create new element with class .widget. Inside it add element with a correct class for a widget you want to use. You can also add heading with a .widget-title class inside widget element to add heading line to the widget.

Flickr Feed

Vistous template uses Joel's Sutherland jQuery plugin for getting images from Flickr feed. You can read more about that plugin in this blog post

<div class="widget" id="widget-flickr">

    <h5 class="widget-title">Flickr Feed</h5>

    <div class="widget-flickr lightbox-gallery" data-plugin-options='{"delegate": "a", "type": "image", "gallery": {"enabled": true}}'>

        <ul class="flickr-feed" data-plugin-options='{"qstrings": {"id": "44802888@N04"}}'>
        </ul>

    </div>

</div>

Example Flickr widget configuration. This code will display content from Flickr account of ID '44802888@N04' and create lightbox gallery for the images.

Twitter Feed

Vistous uses Twitter Feed jQuery plugin for displaying tweets (https://github.com/sonnyt/Tweetie). You need a web server with PHP installed in order for Twitter Feed plugin to work. Before using plugin you need to set up your consumer keys and access tokens in config.php file. This file is located in php folder under template's root folder. You can get your keys here by creating new Twitter Application.

Important!

This plugin was slightly modified to better work with Vistous. To ensure complete compability with template, do not replace this plugin by its original version.

<div class="widget" id="widget-twitter">

    <h5 class="widget-title">Twitter Feed</h5>

    <div class="widget-twitter">

        <ul class="twitter-feed" data-plugin-options='{"username": "envato", "count": "3"}'>
            <li></li>
        </ul>

    </div>

</div>

Plugin options:
  • username - twitter account to load tweets from.
  • hashtag - load tweets with a specific hashtag.
  • hideReplies - set true if you want to hide "@" replies as well. Or leave it false to just to show your tweets and no replies.
  • count - number of tweets to display.
  • apiPath - path to your tweet.php file. There shouldn't be a need for you to change this.
  • dataFormat - date format, leave unchanged when using timestamp.
  • timestamp use timestamp, false to display date of a tweet.
Instagram Feed

Vistous uses Instafeed jQuery plugin to display your Instagram photos. To use the plugin you will need valid client Id from Instagram's API. You can register to get one on Instagram's website.

<div class="widget" id="widget-instagram">

    <h5 class="widget-title">Instagram Feed</h5>

    <div class="widget-instagram">

        <div class="instagram-feed" data-plugin-options='{"get": "user", "userId": "528817151"}'>
        </div>

    </div>

</div>

Basic plugin options:
  • clientId - your API client ID from Instagram.
  • accessToken - A valid oAuth token. Can be used in place of a client ID.
  • get - what the plugin should fetch.
    • popular - images from the popular page.
    • tagged - images with a specific tag. Use tagName option to specify the tag.
    • location - images from a location. Use locationId option to specify the location.
    • user - images with a user. Use userId option to specify the user.
  • limit - maximum number of images to add.
  • sortBy - sort the images in a set order (none, most-recent, least-recent, most-liked, least-liked, most-commented, least-commented, random).
  • links - should the images be wrapped with a link to the photo on Instagram (boolean).
  • resolution - size of images to get (thumbnail, low_resolution, standard_resolution).

You can see the full list of options at the plugin's website.

Elements

Accordion

The general structure for accordion is as follows.

<ul class="accordion">

    <li>
        <div class="title">
            Accordion Panel Title
        </li>

        <div class="content">
            <p>Accordion Panel Content</p>
        </div>
    </li>

</ul>

This will result in "plain" style accordion that allows multiple panels opened at the same time. To make one-at-the-time accordion, just remove the .accordion class from the list element and add data-plugin-accordion-single attribute.

Available classes:
  • .ac-button-dark - "Button" style accordion. Active panel title has dark background.
  • .ac-button-color - "Button" style accordion. Active panel titlte has theme color background.
Alert Boxes

For detailed information about this element refer to Bootstrap documentation. As addition to the Bootstrap's features, Vistous include some styling classes

Available classes:
  • .alert-bg-light - Alert box with light gray background.
  • .alert-bg-gray - Alert box with gray background.
  • .alert-bg-dark - Alert box with dark background.
  • .alert-bg-color - Alert box with theme color background.
  • .alert-border-color - Alert box with transparent background and theme color border.
  • .alert-info-border - Bootstrap's info alert but bordered.
  • .alert-success-border - Bootstrap's success alert but bordered.
  • .alert-warning-border - Bootstrap's warning alert but bordered.
  • .alert-danger-border - Bootstrap's danger alert but bordered.
Buttons

Vistous template include complex button system. All styling is done by adding CSS classes to a button element.

Size classes:

There are 5 pre-defined button sizes. Where there is no sizing class, button will have default (middle) size.

  • .btn-xsmall -
  • .btn-small -
  • .btn-large -
  • .btn-xlarge -
  • .btn-fullwidth - Button will span through all of it's parent width.
Background classes:
  • .btn-bg-transparent - transparent background. Nothing else is altered.
  • .btn-bg-gray - gray background with white text.
  • .btn-bg-dark - dark background with white text.
  • .btn-bg-light - light gray background with dark text.
  • .btn-bg-color - theme color background with white text.
  • .btn-bg-red - red background with white text.
  • .btn-bg-turquoise - turquoise background with white text.
  • .btn-bg-green - green background with white text.
  • .btn-bg-blue - blue background with white text.
  • .btn-bg-purple - purple background with white text.
  • .btn-bg-yellow - yellow background with white text.
  • .btn-bg-orange - orange background with white text.

All background classes (except .btn-bg-transparent) have their default hover styles, that can be changed using hover classes (more info below).

Border classes:

Use these classes if you want your button to be "bordered" (i.e. have transparent background and colored border).

  • .btn-border-gray - gray background with light text.
  • .btn-border-dark - dark background with dark text.
  • .btn-border-light - light background with middle text.
  • .btn-border-color - theme color background and text.
  • .btn-border-red - red background and text.
  • .btn-border-turquoise - turquoise background and text.
  • .btn-border-green - green background and text.
  • .btn-border-blue - blue background and text.
  • .btn-border-purple - purple background and text.
  • .btn-border-yellow - yellow background and text.
  • .btn-border-orange - orange background and text.
Text color classes:

You can also specify custom text color for all button styles. You just need to add one of following classes to your button element.

  • .btn-text-gray
  • .btn-text-dark
  • .btn-text-light
  • .btn-text-color
  • .btn-text-red
  • .btn-text-turquoise

Pie Charts

<div class="pie-chart" data-percent="95" data-plugin-options='{"barColor": "#e74c3c"}'>
    <span class="pc-label text-middle">Pie Chart Label</span>
</div>
Plugin options:
  • accX - animation will fire when element in viewable area with accX px accuracy in X axis.
  • accY - animation will fire when element in viewable area with accX px accuracy in Y axis.
  • barColor - the color of the curcular bar. You can either pass a valid css color string, or a function that takes the current percentage as a value and returns a valid css color string.
  • trackColor - the color of the track, or false to disable rendering.
  • scaleColor - the color of the scale lines, false to disable rendering.
  • lineCap - defines how the ending of the bar line looks like. Possible values are: butt, round and square.
  • lineWidth - width of the chart line in px.
  • size - size of the pie chart in px. It will always be a square.
  • displayPercent - display current bar position in percent, false to disable rendering.
  • animate - Object with time in milliseconds and boolean for an animation of the bar growing ({ duration: 1000, enabled: true }), or false to deactivate animations.
